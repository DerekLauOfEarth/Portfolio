Random Spawning System (Unity Engine)
(2021- 2022)

A 3D random spawning system inspired by Frogger.

Project is structured with:

- ScriptableObject game architecture

- Object Pooling spawning optimisation

Most of the project's GameObject settings are stored within multiple ScriptableObjects which ScriptableObject's modularity make it easier to compartmentalise each stage of the GameObject-data relationship.

Further more, by storing data within ScriptableObject, it offers opportunity for rapid configuration in which its value/ data can be easily slotted in/ out or having multiple copies of similar nature but different value ScriptableObjects.

As a result, I can have multiple copies of ScriptableObject containing different set of spawn rate configuration, and can easily swap the spawn rate setting during runtime in which any change of value can also persist beyond runtime.

With ObjectPooling, the spawning process has been optimised by reducing the overhead of constant instantiation of GameObject with a set of inactive pre-instantiated of GameObjects to be activated only when needed.
